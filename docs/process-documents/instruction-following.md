# 指令跟随评测流程

## 概述

指令跟随（Instruction Following）是评估大语言模型能力的重要维度。本流程旨在系统化地评测模型对各类指令的理解和执行能力。

## 评测目标

1. 评估模型对简单指令的理解
2. 评估模型对复杂指令的处理
3. 评估模型对多步骤任务的支持
4. 检测模型的指令遵循率

## 详细步骤

### 步骤1: 数据集准备

#### 数据收集
- 收集多样化指令样本
- 覆盖不同难度级别
- 确保数据质量

#### 数据分类
```
简单指令: 单步操作
复合指令: 多步操作
约束指令: 包含特定约束
创意指令: 开放性任务
```

### 步骤2: 评测执行

#### 自动化评测
```python
# 评测代码示例
def evaluate_instruction_following(model, dataset):
    results = []
    for sample in dataset:
        response = model.generate(sample.instruction)
        score = calculate_score(response, sample.expected)
        results.append(score)
    return aggregate_results(results)
```

#### 评测指标
- 指令遵循率
- 任务完成率
- 格式准确率
- 质量评分

### 步骤3: 结果分析

#### 统计分析
- 总体得分
- 各维度得分
- 错误类型分布

#### 案例分析
- 成功案例
- 失败案例
- 边界案例

## 注意事项

1. **数据多样性** - 确保指令类型覆盖面广
2. **评分标准** - 制定清晰、客观的评分规则
3. **人工复核** - 关键结果需要人工验证
4. **迭代优化** - 根据结果持续改进评测流程

## 最佳实践

- 使用多维度评分而非单一指标
- 定期更新评测数据集
- 结合自动化和人工评估
- 保持评测流程的可复现性
